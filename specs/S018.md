# S018: Onedir PyInstaller packaging

## Title
Onedir PyInstaller packaging

## Spec Summary
Move packaging guidance and scripts from the onefile approach to an onedir build that starts faster and follows repo ADRs.

## Spec Goal
Provide a repeatable onedir packaging flow with documentation updates so contributors can build and run a standalone CLI bundle.

## DoD checklist
- [x] Add PyInstaller as a dev dependency in `pyproject.toml` with `uv` sync instructions
- [x] Add an onedir PyInstaller spec file (e.g., `git_control_onedir.spec`) using the playbook guidance
- [x] Add `scripts/build_dist_onedir.sh` as the canonical onedir build script (root script per ADR-004)
- [x] Update `scripts/README.md` to document the new build script (ADR-007)
- [x] Update `README.md` and relevant docs (new `docs/packaging.md` or existing docs) to reflect onedir build/run steps and remove any onefile guidance
- [x] Update `specs/plan.md`
